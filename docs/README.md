# はじめに

GPT-2 の推論パイプラインを理解した次のステップとして、画像生成 AI の仕組みを学びましょう。Stable Diffusion 1.5 (SD 1.5) は、テキストから画像を生成する拡散モデルの代表的なアーキテクチャです。

GPT-2 が「次のトークンを予測する」ことでテキストを生成したように、SD 1.5 は「ノイズを少しずつ除去する」ことで画像を生成します。内部では Attention や残差接続など GPT-2 と共通する概念が多く使われていますが、画像を扱うために畳み込み (Conv2D) やグループ正規化 (GroupNorm) といった新しい概念も登場します。

このプロジェクトでは、SD 1.5 の推論パイプラインを PyTorch の基本演算からスクラッチ実装し、「動くコードを見てから仕組みを深掘りする」という流れで学習を進めます。GPT-2 の知識を土台にしているため、「[GPT-2 推論エンジン入門](https://github.com/7shi/my-gpt2/tree/main/docs)」を先に読んでおくことを推奨します。

なお、CPU 実行を前提としており、実用的な速度で生成できるのは 256×256 が上限です。SD 1.5 は 512×512 で訓練されているため 256×256 ではまともな画像が生成できませんが、256×256 向けにチューニングされた miniSD では比較的良好な結果が得られます。PyTorch は NumPy の約 10 倍高速なため採用していますが、原理を説明する目的から `F.conv2d` 等の高レベル関数は使わず、NumPy と同水準の書き方に留めています。

## 構成

SD 1.5 の推論パイプラインを、データの流れに沿って分解します。

テキストはまず **CLIP Text Encoder**（GPT-2 と類似の Transformer）によって条件ベクトルに変換されます。次にランダムノイズから出発し、**U-Net** が条件ベクトルを参照しながらノイズを予測、**DDIM Scheduler** がノイズを除去するステップを 10 回繰り返します。最後に **VAE Decoder** が潜在表現をピクセル画像に変換します。

各章では、この流れの一区間を取り出し、スクラッチ実装のコードを動かしながら、「なぜその計算をするのか」を確認していきます。パイプラインの解説後は、学習済みモデルを少量のパラメータで適応させる **LoRA** と、それを応用して 2 ステップで画像を生成する **LCM LoRA** を扱います。
